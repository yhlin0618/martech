## 根本原因分析

### 原始分群邏輯的問題

**舊方法**: 使用固定百分位數（P20/P80）切分

```r
# 舊的 F value 分群
p80 <- quantile(df$tag_010_rfm_f, 0.8, na.rm = TRUE)
p20 <- quantile(df$tag_010_rfm_f, 0.2, na.rm = TRUE)

f_segment = case_when(
  tag_010_rfm_f >= p80 ~ "高頻買家",
  tag_010_rfm_f >= p20 ~ "中頻買家",
  TRUE ~ "低頻買家"
)
```

**問題場景**:
1. **高單次購買率**: 當90%客戶只購買1次時
   - F值集中在1附近
   - P80可能= 1, P20可能= 1
   - 結果：大部分客戶都被歸為同一組

2. **低變異度M值**: 當大部分客戶消費金額接近時
   - M值集中在均值附近
   - P80和P20非常接近
   - 結果：中消費買家幾乎沒有

### 3. M Value分群改善

**策略**: 智能分群 - 檢測變異度，動態選擇切分方法

```r
# 檢查變異度
m_cv <- sd(df$tag_011_rfm_m) / mean(df$tag_011_rfm_m)
m_median <- median(df$tag_011_rfm_m)
m_p20 <- quantile(df$tag_011_rfm_m, 0.2)
m_p80 <- quantile(df$tag_011_rfm_m, 0.8)

# 判斷是否為低變異度
if (m_cv < 0.2 || (m_p80 - m_p20) / m_median < 0.3) {
  # 情境A: 低變異度 → 使用均值±標準差切分
  m_mean <- mean(df$tag_011_rfm_m)
  m_sd <- sd(df$tag_011_rfm_m)
  threshold_low <- m_mean - 0.5 * m_sd
  threshold_high <- m_mean + 0.5 * m_sd

  m_segment = case_when(
    tag_011_rfm_m > threshold_high ~ "高消費買家",
    tag_011_rfm_m >= threshold_low ~ "中消費買家",
    TRUE ~ "低消費買家"
  )
} else {
  # 情境B: 正常分布 → 使用P33/P67切分
  p67 <- quantile(df$tag_011_rfm_m, 0.67)
  p33 <- quantile(df$tag_011_rfm_m, 0.33)

  m_segment = case_when(
    tag_011_rfm_m >= p67 ~ "高消費買家",
    tag_011_rfm_m >= p33 ~ "中消費買家",
    TRUE ~ "低消費買家"
  )
}
```

**判斷條件**:
1. **CV < 0.2**: 變異係數小於0.2（低變異度）
2. **(P80-P20)/Median < 0.3**: 百分位數範圍過窄

**優點**:
- **情境A**: 均值±標準差確保即使在低變異度下也能區分三組
- **情境B**: 百分位數確保三組數量均衡
- 自動適應數據分布特性

## 技術說明

### 為什麼不能只用P33/P67？

雖然P33/P67能保證三組數量均衡，但在極端數據分布下會產生不合理的分類：

**範例**: 1000個客戶，950人購買1次，50人購買2次以上

- **P33 = 1, P67 = 1** (因為67%的人都是1次)
- 結果：無法區分前67%的客戶

**解決方案**: 使用固定閾值（1次、2次、3+次）確保有意義的分組

---

### 為什麼M值要用均值±標準差？

當M值變異度低時（如大部分客戶消費金額接近$30）：

- **P33 = $29, P67 = $31** (太接近了)
- 使用均值±0.5SD能創造有意義的區間
  - 均值 = $30, SD = $5
  - 低消費: < $27.5
  - 中消費: $27.5 - $32.5
  - 高消費: > $32.5

## 參考資料

### 統計學原理
- **變異係數 (CV)**: SD / Mean，衡量數據離散程度
  - CV < 0.15: 變異度很低
  - CV 0.15-0.3: 變異度中等
  - CV > 0.3: 變異度高

- **百分位數 (Percentile)**:
  - P20/P80: 傳統RFM切分點（20%-20%-60%分組）
  - P33/P67: 均等三分位（33%-34%-33%分組）

### 業務邏輯
- **單次購買率**: 電商常見指標，>70%表示留存率低
- **低變異度M值**: 可能表示固定價格商品或價格帶單一
