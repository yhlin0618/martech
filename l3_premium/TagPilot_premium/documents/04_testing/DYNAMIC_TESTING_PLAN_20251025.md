**測試範圍**: 全部 6 個模組的功能驗證

## 📋 測試概覽

### 測試目標
1. 驗證所有模組在實際運行環境中的功能正確性
2. 確認用戶界面的互動性和響應性
3. 驗證數據處理邏輯的準確性
4. 測試錯誤處理和邊界情況
5. 確認視覺化圖表的正確顯示

### 測試環境
- **R 版本**: 4.x
- **Shiny 框架**: bs4Dash
- **測試資料**: test_data/sample_customer_data.csv
- **執行方式**: 本地 Shiny Server

### 測試資料需求
```r
# 必填欄位
- customer_id: 客戶唯一識別碼
- transaction_date: 交易日期 (YYYY-MM-DD)
- transaction_amount: 交易金額 (數值)

# 建議資料量
- 最少: 100 筆客戶記錄
- 建議: 1,000+ 筆客戶記錄
- 理想: 10,000+ 筆客戶記錄
```
