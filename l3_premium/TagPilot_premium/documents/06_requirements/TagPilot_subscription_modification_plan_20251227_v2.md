## 詳細修改規格

### 1️⃣ 平均購買週期 NA 處理（🔴 緊急）

**檔案**: `modules/module_customer_value_analysis.R`, `modules/module_customer_export.R`

**問題描述**:
- F值=1（只購買1次）的客戶，平均購買週期顯示為 1 天，不合理
- 應顯示 NA，因為單次購買無法計算購買週期

**修改內容**:
```r
# 當 F值 = 1（購買次數 = 1）時
# 平均購買週期應顯示 NA
ipt_display = ifelse(ni == 1 | f_value == 1, NA, ipt_mean)
```

**影響範圍**:
- [ ] UI 顯示：顧客價值分析頁面
- [ ] CSV 輸出：客戶標籤輸出表

**來源**: `TagPilot訂閱板修改建議_20251223.pdf` 第 7-8 頁

| 決策順序 | 判斷條件 | 主策略 | 行銷目的 | 行銷方案建議（完整版） |
|---------|---------|--------|---------|----------------------|
| ① | NES ∈ {S1,S2,S3} | 喚醒/回流 | 防流失 | 1)7天內只接觸1次(EDM/LINE/簡訊擇一) 2)主題：「回來看看你可能用得到的」3)內容：一句關懷+一個使用提醒+一個CTA 4)CTA：查看/了解/回顧(不成交) 5)14天無互動→最後一次回流誘因 6)再無反應→低頻名單 |
| ② | CAI低 + 靜止戶=中/高 | 關係修復 | 降反感 | 1)推播降頻30-50%(停促銷2週) 2)改發資訊型內容(知識/技巧/案例) 3)每次只放1個重點 4)非銷售互動(投票/問一題) 5)觀察2-4週再重新判斷 |
| ③ | 靜止戶=高 | 成本控管 | 避免浪費 | 1)設為低維運名單 2)年/半年一次存在感訊息(不賣) 3)CLV高者保留人工關懷入口 4)有互動才回到①或② |
| ④ | NES=N (新客) | Onboarding | 建立信任 | 1)7-14天3次流程 2)第一次：先完成1件事 3)第二次：不踩雷指南 4)第三次：低風險下一步 5)明示可退訂/不打擾 |
| ⑤ | RFM≤5 + CLV低 | 低成本培養 | 不放棄 | 1)低成本通路(EDM/站內) 2)入門選項(低單價/低複雜) 3)內容+軟誘因 4)被動觸發才跟進 5)KPI以互動為主 |
| ⑥-1 | 5<RFM≤10 + 穩定度低 | 標準培養(保守) | 建立穩定節奏 | 1)用「情境推薦」：依他買過/用過的情境，推薦1-2個延伸選擇 2)搭「信任證據」：評價、案例、同類人怎麼選 3)只做「產品推薦」：降低選擇成本，不推升級、不推高額組合 4)先讓他形成固定購買/固定使用節奏 |
| ⑥-2 | 5<RFM≤10 + 穩定度中 | 標準培養(核心) | 擴展需求 | 1)用「情境推薦」2)搭「信任證據」3)做「交叉銷售」：延伸場景/搭配需求 4)設「輕任務」促進第二品類嘗試 |
| ⑥-3 | 5<RFM≤10 + 穩定度高 | 標準培養(進階) | 提升客單 | 1)用「情境推薦」2)搭「信任證據」3)做「輕向上銷售(soft upsell)」：升級版/加量版/更省時省力的方案 4)用非價格誘因(優先名額/加值服務/專屬權益) |
| ⑦-1 | RFM高 + 穩定度低 | VIP維繫 | 提升穩定度 | 1)給「VIP專屬內容/權益」：新消息、搶先看、優先權 2)推「高毛利交叉」3)先固定節奏(定期回購/定期使用)，暫不推高價升級 |
| ⑦-2 | RFM高 + 穩定度中 | VIP維繫 | 維持滿意度 | 1)給「VIP專屬內容/權益」2)推「高毛利交叉」3)以「推薦+交叉」為主，向上銷售採小步測試 |
| ⑦-3 | RFM高 + 穩定度高 | VIP維繫 | 深化忠誠 | 1)給「VIP專屬內容/權益」2)推「高毛利交叉」3)正式啟動向上銷售 4)同步做轉介紹/推薦 |
| ⑧ | NES=E0 + CLV高 | 尊榮維繫 | 穩定關係 | 1)降低銷售比例(價值/關係為主) 2)老朋友回饋、意見徵詢 3)共創/內測/搶先體驗 4)分級待遇(服務/優先權) |
| ⑨ | 其他/資料不足 | 基礎維繫 | 周延覆蓋 | 1)定期品牌內容/更新 2)讓顧客選偏好 3)有行為再回到①-⑧ |

### 5️⃣ 顧客活躍度 ni>=4 篩選

**檔案**: `modules/module_customer_activity.R`

**問題描述**:
- CAI 計算需要 ni >= 4 筆交易資料
- 目前顯示 ni=2, 3 的客戶活躍度，不合理

**修改內容**:
```r
# 篩選 ni >= 4 的客戶才顯示活躍度
filtered_data <- data %>%
  filter(ni >= 4)

# ni < 4 的客戶，活躍度顯示 NA
activity_display = ifelse(ni < 4, NA, cai_value)
```
