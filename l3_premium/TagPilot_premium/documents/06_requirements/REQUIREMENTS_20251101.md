## 第1部分：資料上傳說明 (Data Upload Instructions)

### 需求 #1.1: 更新資料需求說明

**位置**: 資料上傳頁面

**當前狀態**: 需要更新說明文字

**需求變更**:

#### 資料需求 (紅字標示部分需修改)
```
• 請上傳 至少 1~3 年 的顧客交易數據
  - 資料檔須包含顧客 ID、購買時間和購買金額

• 可同時上傳多個月份檔案
  - 建議 12-36 個月

• 檔案數上限：36 個檔案（3 年資料）

• 檔案格式：CSV（推薦）、XLSX
```

#### 必填欄位說明

**1. ID 或 Email** - 識別顧客的代碼
- 系統可識別：`customer_id`, `email`, `buyer_email`

**2. purchase_time** - 顧客購買產品的時間點
- 格式須符合 EXCEL 規範，如 YYYY-MM-DD
- 系統可識別：`payment_time`, `purchase_time`, `purchase_date`, `date`

**3. price** - 購買產品所花費的金額（數值型態）
- 系統可識別：`lineitem_price`, `amount`, `price`

#### 重要提醒文字

**提醒 1**:
```
💡 系統會自動偵測並對應以上欄位，支援多種命名格式；
當系統無法讀取上述變數時，通常是因為顧客 ID、購買時間和購買金額
這三個變數名稱無法被識別。您可以根據這三個變數的英文名稱進行修改，
這樣系統就能正確對應和識別。
```

**提醒 2**:
```
💡 提醒：price 係指顧客購買產品所花費的金額，
若為跨境電商交易，須注意幣值需換算和統一。
```

**實施優先級**: 中
**實施難度**: 低
**預估工時**: 1小時

## 第2部分：側邊欄選單重組 (Sidebar Menu Restructure)

### 需求 #2.1: 重新組織分析模組結構

**當前結構** (需要修改):
- 資料上傳
- Value × Activity 九宮格
- 客戶基礎價值
- RFM 價值分析
- 客戶狀態
- 生命週期預測

**新結構** (目標架構):

```
(1) 資料上傳

(2) 顧客價值
    └─ 最近購買日、購買頻率、購買金額、RFM 分析

(3) 顧客活躍度
    └─ CAI 分析

(4) 顧客動態
    └─ 顧客狀態、顧客流失風險、顧客入店資歷

(5) 顧客價值與動態市場區隔分析
    └─ 取代之前的 Value × Activity 九宮格分析

(6) 顧客基礎價值
    └─ 顧客購買週期、過去價值、客單價

(7) 顧客生命週期預測
    └─ 靜止戶預測、下次回購時間、交易穩定度、顧客終生價值
    └─ 如果塞不下，可將顧客終生價值往下移成：(8) 顧客終生價值
```

**實施優先級**: 高
**實施難度**: 中
**預估工時**: 4-6 小時
**相關檔案**:
- `app.R` (sidebar 定義)
- `modules/module_*.R` (各模組對應調整)
