### DIFF-001: 新客定義調整 🟡

**補充資料原始定義** (`TagPilot_Lite高階和旗艦版_20251021.md` Line 101)：
```
新客是在平均購買週期內才算新客
```

**程式碼邏輯**（補充資料）：
```r
ni == 1 & customer_age_days <= avg_ipt ~ "newbie"
```

**實際實現**：
```r
ni == 1 & customer_age_days <= 60 ~ "newbie"  // 固定60天窗口
```

#### 修改理由（技術與業務）

**問題診斷**：
1. **邏輯矛盾**：
   - `avg_ipt` 基於**已回購客戶**計算（ni ≥ 2）
   - 平均值約 20-30 天（活躍回購客戶的間隔短）
   - 新客首購後通常需 60-90 天才決定回購
   - 結果：真實新客 (30-60天) > avg_ipt (20天) → 被誤判為沉睡客

2. **測試資料驗證**：
   ```
   測試資料 1（sample_customer_data.csv）：
   - avg_ipt: 234.25天，新客數: 0 位（0%）❌

   測試資料 2（realistic_customer_data.csv）：
   - avg_ipt: 21.58天，新客數: 0 位（0%）❌

   修改後結果：
   - 60天窗口，新客數: 135 位（13.5%）✅
   ```

3. **業務影響**：
   - 無法識別新客 → 無法執行新客策略（首購優惠等）
   - 用戶體驗差：看到「新客數：0」

#### 採用方案與優勢

**方案選擇**：固定60天窗口

**優勢**：
1. ✅ 符合業務常識（行業標準）
2. ✅ 穩定可預測（不受資料分佈影響）
3. ✅ 測試驗證有效（13.5% 新客率）
4. ✅ 易於理解和解釋

**無負面影響**：
- ✅ 不影響其他生命週期階段判定
- ✅ 不影響整體分析邏輯
- ✅ 符合 MP029（基於真實資料調整）

**詳細分析**：[GAP001_NEWBIE_DEFINITION_ANALYSIS.md](GAP001_NEWBIE_DEFINITION_ANALYSIS.md)

### DIFF-002: 低頻客戶處理策略 🟢

**補充資料原始要求** (`TagPilot_Lite高階和旗艦版_20251021.md` Line 85)：
```
理論上,交易次數少於四次的顧客沒辦法算活躍度,故將低於四次交易的顧客刪除。
```

**原始邏輯**：刪除 ni < 4 的客戶
```r
df <- df %>% filter(times >= 4)
```

**實際實現**：保留 + 降級策略
```r
# 交易次數 >= 4：使用 CAI（標準計算）
activity_level = case_when(
  ni >= 4 ~ quantile_based_CAI(...),

  # 交易次數 < 4：使用 Recency 降級（保留客戶）
  ni < 4 ~ quantile_based_Recency(...),

  TRUE ~ "中"
)
```

#### 修改理由（符合 MP029 原則）

**問題診斷**：
1. **資料完整性損失**：
   - ni < 4 的客戶佔比 20-40%
   - 刪除 = 丟失大量商業洞察

2. **違反 MP029 原則**：
   - **MP029: No Fake Data Principle** - 不生成假資料，不刪除真實資料
   - 所有客戶都是真實資料，都有分析價值

3. **業務價值流失**：
   - 低頻客戶代表潛在機會或流失風險
   - 需要差異化策略（非刪除）

#### 降級策略設計

**技術實現**：
| 客戶類型 | 活躍度計算 | 統計意義 | 百分位數 |
|---------|-----------|----------|---------|
| ni ≥ 4 | CAI | 購買行為加速/減速趨勢 | P20/P80（CAI 分佈） |
| ni < 4 | Recency | 最近購買時間 | P20/P80（R 值分佈） |

**關鍵設計**：
- ✅ 分群獨立計算百分位數
- ✅ 不同統計意義（CAI看趨勢，R看狀態）
- ✅ 保留所有客戶
- ✅ 文檔說明差異

#### 方案比較

| 方案 | ni < 4 處理 | 客戶覆蓋率 | 資料遺失 | MP029 合規 | 業務價值 |
|------|------------|-----------|---------|-----------|---------|
| **刪除**（補充資料） | 移除 | 60% | 40% | ❌ | ❌ 部分流失 |
| **降級**（採用） | 保留+替代指標 | 100% | 0% | ✅ | ✅ 完整保留 |

**優勢**：
- ✅ 符合 MP029 原則
- ✅ 提升客戶覆蓋率 40%
- ✅ 更完整的客戶視圖
- ✅ 更豐富的行銷策略

**無負面影響**：
- ✅ CAI 計算準確度不受影響（ni≥4 仍用 CAI）
- ✅ 統計意義清晰（文檔說明）
- ✅ 用戶體驗改善（不會看到客戶消失）

**詳細分析**：[COMPREHENSIVE_COMPLIANCE_AUDIT_20251026.md](COMPREHENSIVE_COMPLIANCE_AUDIT_20251026.md) (DIFF-002 章節)

#### 4.1 CAI 計算邏輯

**CAI (Customer Activity Index)**:
- [ ] 公式：`CAI = (mle - wmle) / mle`
- [ ] 其中：
  - `mle`：MLE 估計的購買週期（IPT）
  - `wmle`：加權 MLE 估計
  - `ni`：客戶購買次數（Frequency）

**計算條件**:
- [ ] 僅計算 `ni ≥ 4` 的客戶（購買 4 次以上）
- [ ] `ni < 4` 的客戶使用降級策略：
  - 使用 Recency 作為活躍度指標
  - 或標示為「樣本不足」

#### 4.2 生命週期階段定義

**5 個階段**:

1. [ ] **newbie**（新客戶）
   - 條件：F = 1 且 R < IPT_median
   - 描述：第一次購買且在觀察期內

2. [ ] **active**（活躍客戶）
   - 條件：CAI < Recency 且 Recency < 2×IPT
   - 描述：活躍且預期會再次購買

   - 條件：2×IPT ≤ Recency < 3×IPT

4. [ ] **sleepy**（睡眠客戶）
   - 條件：3×IPT ≤ Recency < 4×IPT
   - 描述：高流失風險

5. [ ] **dormant**（休眠客戶）
