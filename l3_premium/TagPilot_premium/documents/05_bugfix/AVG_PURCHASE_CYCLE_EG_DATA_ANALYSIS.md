# EG 測試資料平均購買週期為 1 的說明

## 背景
- 使用 `test_data/eg` 進行平均購買週期驗證時，前端卡片顯示 1 天。
- 已在程式邏輯中排除單次購買顧客（`ni >= 2` 才計算）。

## 驗證步驟與結果
- 以 R 腳本讀取 `test_data/eg` 下 4 份 CSV，欄位標準化為 `customer_id / transaction_date / transaction_amount`。
- 基本統計：
  - 總交易 39,930 筆，顧客數 38,350。
  - 單次購買佔比 **96.5%**（`ni == 1`）。
- 對回購顧客（`ni >= 2`）計算：`ipt_span = last_purchase - first_purchase`，並使用既有邏輯 `ipt = pmax(ipt_span, 1)`；平均購買週期 = `ipt / (ni - 1)`。
  - `avg_cycle` 中位數 = **1 天**。
  - 原因：少數回購顧客的多筆交易集中在同一天或隔天，`last-first` 變成 0 或 1，被 `pmax` 壓到 1。

## 解釋
- 顯示為 1 並非未排除單次購買顧客，而是 **回購顧客樣本太少且交易集中在同日/隔日**，造成週期被壓縮。
- 現行邏輯：排除了 `ni == 1`，但同日多筆仍會導致 `ipt = 1`。

## 若要改善顯示的選項（擇一）
1) **改用相鄰交易日間隔的中位數**（忽略同日多筆），不對 `ipt` 做 `pmax(…, 1)`，讓 0 間隔保留為 0，再取中位數/均值。
2) **設定下限前先過濾同日交易**：同日多筆先合併或視為 0 間隔，再計算 `ipt / (ni - 1)`；必要時可對最終結果加下限。
3) **顯示時用分位數或 trimmed mean**：在卡片上改用 `median(cycle)` 並對極小值設閾值，如 `<1 天` 顯示為 `≤1 天`。
