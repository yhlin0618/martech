### DIFF-001 詳細說明：新客定義調整

**問題診斷**:
```
原定義: ni == 1 AND customer_age_days <= avg_ipt
結果: 測試資料中 0 位新客 (0%)
原因: avg_ipt = 21.58天 (回購客戶平均間隔)
      但最年輕的單次購買客戶已30天
      → 30天 > 21.58天 → 不符合條件
```

**解決方案**:
```
調整後: ni == 1 AND customer_age_days <= 60
結果: 預期 135 位新客 (13.5%)
理由: 新客需60-90天決策週期（業務現實）
      固定窗口更穩定、符合行業標準
```

**原則檢查**: ✅ 符合
- ✅ MP029: 適應真實資料特性，非創建假資料
- ✅ 業務導向: 符合實際購買行為
- ✅ 行業標準: 大多數CRM使用30-90天固定窗口

**文檔記錄**: ✅ 已有完整分析
- GAP001_NEWBIE_DEFINITION_ANALYSIS.md (407行)
- 包含5種方案對比、測試預測、決策流程

**待辦事項**: 更新Work_Plan.md Task 4.1說明此調整
