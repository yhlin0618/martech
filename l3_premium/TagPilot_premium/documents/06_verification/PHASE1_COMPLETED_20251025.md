### 1.2 檔案數量限制檢查

**檔案**: `modules/module_upload.R` Line 120-141

**新增功能**:

```r
# 1. 上限檢查（36個檔案）
MAX_FILES <- 36
if (nrow(files) > MAX_FILES) {
  顯示錯誤：「檔案數量超過上限！已上傳X個檔案，但系統最多只能處理36個檔案」
  return()
}

# 2. 數量提示
if (nrow(files) >= 12) {
  顯示成功：「已選擇X個檔案（約Y年資料），符合建議」
} else {
  顯示警告：「已選擇X個檔案（不足1年資料），建議至少12個月份」
}
```

**效果**:
- ✅ 防止上傳過多檔案導致系統負擔
- ✅ 提供即時反饋（檔案數量和年份估算）
- ✅ 引導用戶上傳適當數量的資料

### 1.3 CSV匯出格式優化

**檔案**: `modules/module_dna_multi_premium.R` Line 696-772

**修改內容**:

#### 修改前（舊格式）:
匯出所有欄位（15-20個欄位）：
```
customer_id, ni, r_value, f_value, m_value, 
lifecycle_stage, value_level, activity_level,
tag_001, tag_002, ..., tag_038,
insufficient_data_flag
```

#### 修改後（新格式）:
**只匯出6個核心欄位**：
```
客戶ID, 價值等級, 活躍度等級, 生命週期階段, 策略名稱, 行動方案
```

**實際範例**:
| 客戶ID | 價值等級 | 活躍度等級 | 生命週期階段 | 策略名稱 | 行動方案 |
|--------|---------|-----------|------------|---------|---------|
| cust001 | 高 | 高 | active | 王者引擎 (主力) | VIP社群 + 新品搶先權 |
| cust002 | 中 | 低 | sleepy | 成長停滯 (瞌睡) | Push+SMS雙管齊下 |
| cust003 | 低 | 高 | active | 潛力新芽 (主力) | 引導升級高單價品 |

**其他改進**:
1. 檔名改為 `customer_strategy_YYYY-MM-DD.csv`（更明確）
2. 欄位名稱使用中文（更易讀）
3. 下載按鈕文字改為「下載客戶策略建議」
4. 說明文字改為「包含客戶ID、價值等級、活躍度、生命週期階段及對應的行銷策略建議」

**效果**:
- ✅ 檔案更精簡（從20欄位減少到6欄位）
- ✅ 適合直接用於廣告投放
- ✅ 每個客戶都有明確的策略建議
- ✅ 符合PDF需求
