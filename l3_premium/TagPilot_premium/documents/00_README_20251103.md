## 🎓 Key Learnings

### Technical Insights

1. **Variable Scope Matters**
   - Always initialize variables before conditional branches
   - undefined `icon_name` caused hours of debugging

2. **Parameter Order is Critical**
   - R function calls interpret positional parameters strictly
   - `actionButton(id, label, icon = icon(...))` not `actionButton(id, icon(...), label)`

3. **Identity Mapping Can Fail**
   - Avoid unnecessary `label_map["新客"] = "新客"` transformations
   - Direct column usage more reliable with Chinese characters

4. **Data Quality Thresholds**
   - CV < 0.15 reliably indicates problematic variance
   - >90% single-purchase rate suggests short timespan

5. **User-Friendly Names Win**
   - Chinese column names dramatically improve UX
   - Technical names (`tag_017_*`) confuse business users

### Documentation Best Practices

1. **Before/After Examples**
   - Clear comparisons help users understand changes
   - Code snippets more valuable than prose descriptions

2. **Root Cause Analysis**
   - Document WHY, not just WHAT
   - Helps prevent similar issues in future

3. **Testing Plans**
   - Include expected values for test data
   - Specify exact checkpoints to verify

4. **Cross-References**
   - Link related documents
   - Create navigation paths for readers

### Project Management

1. **Incremental Validation**
   - Validate syntax after every file change
   - Catches errors immediately

2. **Todo List Discipline**
   - Keep todo list updated in real-time
   - Mark completed items immediately

3. **Session Summaries**
   - Comprehensive summaries enable easy continuation
   - Future developers can understand context
