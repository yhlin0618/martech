# InsightForge 故障排除指南

## 🚀 快速啟動

### Windows PowerShell 用戶
```powershell
# 方法 1: 使用啟動腳本（推薦）
.\run_app.ps1

# 方法 2: 直接命令
Rscript -e "shiny::runApp(port = 3838, host = '0.0.0.0')"
```

### 常見啟動錯誤

#### ❌ PowerShell 語法錯誤
**錯誤訊息:** `'&&' 語彙基元不是有效的陳述式分隔符號`

**原因:** PowerShell 語法與 bash 不同

**解決方案:**
```powershell
# ❌ 錯誤寫法
cd InsightForge && Rscript -e "shiny::runApp()"

# ✅ 正確寫法
cd InsightForge
Rscript -e "shiny::runApp(port = 3838, host = '0.0.0.0')"

# ✅ 或使用啟動腳本
.\run_app.ps1
```

## 🔧 資料對應問題

### ❌ "無法建立評分與銷售資料的對應關係"

這個錯誤已經修復！新版本包含智能對應策略：

#### 修復內容：
1. **智能對應策略**: 自動處理變體數量不匹配的情況
2. **調試信息**: 顯示詳細的對應過程
3. **寬容邏輯**: 不再因為數量差異而失敗

#### 使用調試腳本：
```r
# 在 R 或 RStudio 中運行
source("debug_data_mapping.R")
```

調試腳本會檢查：
- ✅ 評分資料是否存在
- ✅ 銷售資料是否存在  
- ✅ 變體名稱對應關係
- ✅ 資料品質和完整性
- ✅ 提供具體的修復建議

### 📊 資料格式要求

#### 評分資料格式：
```
Variation | Score_屬性1 | Score_屬性2 | ...
---------|------------|------------|----
品牌A     | 8.5        | 7.2        | ...
品牌B     | 7.8        | 8.1        | ...
```

#### 銷售資料格式：
```
Variation | Time       | Sales
----------|------------|-------
品牌A     | 2024-01-01 | 1500
品牌B     | 2024-01-01 | 1200
```

### 🔗 對應策略

#### 策略 1: 直接匹配（最佳）
- 評分資料和銷售資料的 `Variation` 名稱完全一致
- 系統自動使用此策略，無需額外設定

#### 策略 2: 智能對應（自動觸發）
- 當變體名稱不匹配時自動啟用
- 按順序配對變體（取較少的數量）
- 顯示配對結果和成功訊息

## 🎯 分析參數調整

### 新增功能：
1. **屬性數量選擇**: 6-10個可調整（原本固定6個）
2. **最小樣本**: 降低至2個觀測值（原本10個）
3. **模型估計**: 更名為"模型估計"（原本"視覺化"）
4. **正向係數**: 只顯示正向影響的屬性

### 使用建議：
- **6-7個屬性**: 適合初步分析
- **8-10個屬性**: 適合深度分析
- **樣本數**: 至少2個，建議10個以上

## 🛠️ 常見問題解決

### 問題 1: 資料上傳失敗
**可能原因:**
- 檔案格式不正確
- 缺少必要欄位
- 資料編碼問題

**解決方案:**
1. 確保 CSV/Excel 格式正確
2. 檢查是否包含 `Variation` 欄位
3. 使用 UTF-8 編碼保存檔案

### 問題 2: 評分失敗
**可能原因:**
- 網路連接問題
- API 配額不足
- 資料格式問題

**解決方案:**
1. 檢查網路連接
2. 減少評分資料筆數
3. 確認資料格式正確

### 問題 3: 分析無結果
**可能原因:**
- 所有係數都是負數
- 資料品質問題
- 變體對應失敗

**解決方案:**
1. 檢查資料品質
2. 使用調試腳本診斷
3. 確認評分和銷售資料的一致性
