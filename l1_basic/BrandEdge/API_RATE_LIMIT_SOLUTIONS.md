# API 速率限制問題解決指南

## 問題描述
當您看到 `❌ API 調用失敗： API 速率限制，已達到最大重試次數` 錯誤時，這表示 OpenAI API 的使用頻率超過了限制。

## 即時解決方案

### 🎯 方案1：使用示例屬性（推薦）
**最快速的解決方法 - 立即可用**

1. 點擊 **「使用示例屬性」** 按鈕
2. 系統會自動載入6個示例屬性：
   - 品質優良
   - 價格實惠  
   - 使用方便
   - 外觀美觀
   - 功能豐富
   - 服務良好

3. 繼續使用 **「開始評分」** 進行分析
4. 系統會使用模擬評分，讓您體驗完整分析流程

**優點：**
- ✅ 無需等待，立即可用
- ✅ 完整的分析體驗
- ✅ 適合學習和測試系統功能

### ⏱️ 方案2：等待重試（自動處理）
**改進的重試機制**

新的系統配置：
- **重試次數：** 5次（原3次）
- **等待時間：** 指數退避 (5, 10, 20, 40, 80秒)
- **最大等待：** 2分鐘
- **進度顯示：** 實時顯示重試進度

當遇到429錯誤時，系統會自動：
```
API 速率限制 (1/5)，等待 5 秒後重試...
API 速率限制 (2/5)，等待 10 秒後重試...
...
```

### 🔄 方案3：稍後重試
**等待一段時間後重新嘗試**

如果重試仍然失敗：
1. **等待 15-30 分鐘**
2. 重新點擊 **「產生 6 個屬性」**
3. API限制通常會在此時間後重置

## API 速率限制說明

### OpenAI API 限制類型
1. **每分鐘請求數 (RPM)**
2. **每分鐘令牌數 (TPM)**  
3. **每日請求數 (RPD)**

### 不同方案的限制
| 方案類型 | RPM | TPM | 狀態 |
|---------|-----|-----|------|
| 免費試用 | 3 | 200 | 非常有限 |
| API 付費 | 3,500+ | 90,000+ | 推薦 |
| Plus 訂閱 | 500 | 40,000 | 適中 |

## 長期解決方案

### 🔧 解決方案A：檢查API設定
1. **確認API金鑰：** 檢查 `.env` 文件中的 `OPENAI_API_KEY`
2. **檢查配額：** 登入 OpenAI 後台查看使用量
3. **升級方案：** 考慮升級到付費方案

### 💳 解決方案B：升級OpenAI方案
**推薦為分析平台使用付費方案：**
- 更高的速率限制
- 穩定的服務品質  
- 支援批量處理

### ⚙️ 解決方案C：優化使用方式
1. **分批處理：** 選擇較少的評論數量（如25-50筆）
2. **錯峰使用：** 避開高峰時段
3. **間隔使用：** 多次分析之間間隔數分鐘
