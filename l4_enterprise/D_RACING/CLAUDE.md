## 開發指令

### Pipeline 操作

```bash
cd scripts/update_scripts

make config-merge          # 合併配置
make run PLATFORM=amz      # 執行 Amazon 平台
make run                   # 執行所有啟用平台
make status                # 檢查狀態
make vis                   # 視覺化 DAG
```

### Shared Git 操作

`scripts/` 下的三個目錄都是 symlink，指向 `shared/` 中的 standalone git repo：

```bash
# 日常更新（直接 commit + push）
cd ../../shared/global_scripts    # 或 update_scripts / nsql
git add -A && git commit -m "變更說明"
git pull --rebase && git push

# 拉取最新（從其他機器同步）
cd ../../shared/global_scripts
git pull
```

> **Note**: 不再使用 `git subrepo`。每個 shared/ 子目錄都是獨立的 git repo，直接用標準 git 操作。
> 有 bug/issue 需追蹤時才用 GitHub Issue + PR 流程。

### 啟動應用

```bash
Rscript app.R              # 本地開發
```
