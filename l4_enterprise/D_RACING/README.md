## Quick Start

### 1. 設定環境變數（必須）

```bash
cp .env.template .env
chmod 644 .env
# 編輯 .env 填入實際憑證
```

**必填憑證：**

| Platform | 環境變數 |
|----------|----------|
| Amazon | `AMZ_ACCESS_KEY`, `AMZ_SECRET_KEY` |
| AI | `OPENAI_API_KEY` |

### 2. 執行 ETL Pipeline

```bash
# 在專案根目錄直接執行
make config-full
make run PLATFORM=amz
```

### 3. 啟動應用程式

```bash
Rscript app.R
# 或用 RStudio 開啟 D_RACING.Rproj
```

### 4. 執行 E2E 測試

```bash
# 安裝一次即可
Rscript -e "install.packages('shinytest2', repos='https://cloud.r-project.org')"

# 從專案根目錄直接執行完整 E2E
make e2e

# 只跑特定測試檔/關鍵字
make e2e-filter FILTER=login
make e2e-filter FILTER=vitalsigns

# 常用快捷指令
make e2e-login
make e2e-vitalsigns
```

E2E 前置條件：
- `data/app_data/app_data.duckdb` 必須存在
- 先把 ETL/DRV 跑完，至少要有 TagPilot / VitalSigns 需要的資料表
- 測試會自動帶入 `NOT_CRAN=true` 與 `E2E_PROJECT_ROOT`


## Shared Git 維護

`scripts/` 下的三個目錄都是 symlink，指向 `l4_enterprise/shared/` 中的 standalone git repo。

```bash
# 日常更新（直接 commit + push）
cd ../../shared/global_scripts    # 或 update_scripts / nsql
git add -A && git commit -m "變更說明"
git pull --rebase && git push

# 拉取最新（從其他機器同步）
cd ../../shared/global_scripts
git pull
```

> 不再使用 `git subrepo`。有 bug/issue 需追蹤時才用 GitHub Issue + PR 流程。
