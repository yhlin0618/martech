## Quick Start

### 1. 設定環境變數（必須）

```bash
# 複製模板
cp .env.template .env
chmod 600 .env

# 編輯 .env 填入實際憑證
```

**必填憑證：**

| Platform | 環境變數 |
|----------|----------|
| Amazon | `AMZ_ACCESS_KEY`, `AMZ_SECRET_KEY` |
| AI | `OPENAI_API_KEY` |

### 2. 測試連線

```bash
# 測試資料庫連線
Rscript scripts/global_scripts/98_test/test_connections.R

# 測試公司識別
Rscript -e '
source("scripts/global_scripts/04_utils/fn_resolve_script_path.R")
print(paste("Company:", get_company_from_config()))
'
```

### 3. 執行 ETL Pipeline

```bash
# 在專案根目錄直接執行
make config-full
make run PLATFORM=amz
```

## Data Source Scope (QEF)

QEF_DESIGN 目前的正式交易資料來源只有 **Amazon (`amz`)**。  
本專案現階段不包含 Cyberbiz (`cbz`) / eBay (`eby`) 的生產 ETL 流程。

- 生產執行請使用：`PLATFORM=amz`
- Amazon `0IM` 已啟用可審計完整性閘門（transactional gate）：
  - 匯入需通過來源覆蓋、列數對帳、關鍵鍵值（如 ASIN）對帳
  - 任一必要檢查失敗即 **rollback**，不允許部分提交

### 4. 啟動應用程式

```bash
# 本地開發
Rscript app.R

# 或用 RStudio 開啟 QEF_DESIGN.Rproj
```

### 5. 執行 E2E 測試

```bash
# 安裝一次即可
Rscript -e "install.packages('shinytest2', repos='https://cloud.r-project.org')"

# 從專案根目錄直接執行完整 E2E
make e2e

# 只跑特定測試檔/關鍵字
make e2e-filter FILTER=login
make e2e-filter FILTER=vitalsigns

# 常用快捷指令
make e2e-login
make e2e-vitalsigns
```

E2E 前置條件：
- `data/app_data/app_data.duckdb` 必須存在
- 先把 ETL/DRV 跑完，至少要有 TagPilot / VitalSigns 需要的資料表
- 測試會自動帶入 `NOT_CRAN=true` 與 `E2E_PROJECT_ROOT`
