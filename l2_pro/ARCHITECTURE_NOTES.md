# L2 Pro 架構說明

## 概述
L2 Pro（專業版）的架構設計尚在規劃中，可能會採用與 L1 Basic 不同的架構模式。

## 可能的架構方向

### 1. 模組化設計
- 將功能拆分為獨立的服務模組
- 模組之間透過 API 通訊
- 支援更靈活的部署配置

### 2. 容器化部署
- 使用 Docker 容器封裝應用
- 支援 Kubernetes 編排
- 便於橫向擴展

### 3. 共享服務層
- 統一的認證授權服務
- 集中式配置管理
- 共享的資料快取層

## Global Scripts 整合策略

L2 Pro 可能採用以下方式之一整合 global_scripts：

1. **微服務封裝**：將 global_scripts 封裝為獨立的服務
2. **NPM/包管理**：透過包管理系統分發
3. **容器映像層**：作為基礎映像的一部分

## 待定事項

- 具體的目錄結構標準
- 服務間通訊協議
- 部署流程規範
- 監控和日誌策略

## 參考資料

- R124 規則目前僅適用於 L1 Basic
- L2 Pro 的標準將根據實際需求另行制定
