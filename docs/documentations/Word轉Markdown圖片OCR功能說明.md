# Word → Markdown 圖片 OCR 功能說明

## 🎯 功能概述

當使用 `/convert` 命令將 Word 文件轉換為 Markdown 時，Claude Code 現在可以自動：

1. 使用 **Pandoc** 提取圖片為獨立檔案
2. 使用 **Claude Code 的視覺能力**讀取並分析圖片內容
3. 自動執行 **OCR**（光學字符識別）或生成描述
4. 將文字內容以可折疊區塊形式插入 Markdown 檔案

## 📋 完整工作流程

### 步驟 1: Pandoc 轉換（自動）

```bash
pandoc input.docx -o output.md --extract-media=./media
```

**結果**：
- ✅ Word → Markdown 轉換
- ✅ 圖片提取到 `media/` 目錄
- ✅ Markdown 中使用相對路徑引用圖片

### 步驟 2: Claude Code 圖片智能分析（自動）

Claude Code 會：

1. **掃描** `media/` 目錄中的所有圖片
2. **讀取**每張圖片（使用 Read tool 的圖片讀取功能）
3. **分析**圖片類型：
   - 📝 **純文字截圖** → 執行 OCR
   - 📊 **圖表/表格** → OCR + 結構化描述
   - 🖼️ **照片/插圖** → 生成詳細描述

### 步驟 3: 更新 Markdown（自動）

在原始圖片引用下方添加 OCR 內容：

```markdown
![圖片標題](media/image.png)

<details>
<summary>📊 圖片內容（OCR 提取）</summary>

[提取的文字內容，保留原始格式和結構]

</details>
```

## 🌟 實際範例

### 原始 Markdown（Pandoc 轉換後）

```markdown
3. 平均購買時間$\mu$：用中位數算，例子說明如下：

![顧客購買間隔計算示意圖](media/image2.png)

**一、**新客：新客的定義是首購＆在平均購買時間內
```

### 處理後的 Markdown（添加 OCR）

```markdown
3. 平均購買時間$\mu$：用中位數算，例子說明如下：

![顧客購買間隔計算示意圖](media/image2.png)

<details>
<summary>📊 圖片內容（OCR 提取）</summary>

### 四、實際例子：

假設你要分析「電動開關器」的購買行為，資料是：

- 最早交易：2024/01/01
- 最晚交易：2024/12/31

👉 **觀察期 = 2024/01/01 ~ 2024/12/31**

接下來：

| 顧客 | 交易日期 | 間隔 |
|------|----------|------|
| A | 2/1、3/1、4/1 | 28、31 |
| B | 1/10、2/15、3/25 | 36、39 |
| C | 3/5、6/10 | 97 |

→ 合併所有間隔 = {28, 31, 36, 39, 97}
→ 中位數 μ₅₀ = 36 天

**因此：**

> 在「觀察期（一年）」內，典型顧客大約每隔 **36 天** 購一次。

**那「新客」的定義就可寫成：**

> 若顧客首購後距今天數 ≤ 36 天 → 新客。

</details>

**一、**新客：新客的定義是首購＆在平均購買時間內
```

## ✨ 優勢

### 1. 🔍 可搜尋性
- ✅ 圖片中的文字變成可搜尋的文本
- ✅ 方便使用 Ctrl+F 查找關鍵字
- ✅ 可以被搜尋引擎索引

### 2. 📱 可存取性
- ✅ 螢幕閱讀器可以讀取內容
- ✅ 提升無障礙性（Accessibility）
- ✅ 符合 WCAG 標準

### 3. 🔄 可編輯性
- ✅ 文字內容可以直接複製貼上
- ✅ 可以修改和更新
- ✅ 不需要重新編輯圖片

### 4. 📊 數據提取
- ✅ 表格數據可以直接使用
- ✅ 數字和統計資料易於引用
- ✅ 可以轉換為其他格式（CSV、Excel）

### 5. 🌐 多語言支援
- ✅ Claude 支援多種語言的 OCR
- ✅ 中文、英文混合內容處理準確
- ✅ 保留數學符號和特殊格式

### 6. 💾 保留原圖
- ✅ 原始圖片保留不變
- ✅ 視覺化和文字內容並存
- ✅ 可折疊區塊不影響閱讀體驗

## ⚙️ 配置選項

在 `/convert` 命令中，你可以指定：

1. **是否啟用 OCR**（預設：啟用）
2. **OCR 語言**（預設：自動偵測）
3. **輸出格式**：
   - 折疊區塊 `<details>` 預設）
   - 直接插入文字
   - 側邊註釋

## 📝 最佳實踐

### ✅ 建議做法

1. **轉換前檢查 Word 檔案**
   - 確保圖片清晰可讀
   - 文字部分最好使用標準字體
   - 表格數據對齊整齊

2. **轉換後驗證**
   - 檢查 OCR 結果準確性
   - 修正可能的錯誤
   - 補充必要的說明

3. **圖片命名**
   - Pandoc 自動生成的圖片名稱（image1.png, image2.png...）
   - 可以在 Markdown 中添加描述性標題

### ⚠️ 注意事項

1. **OCR 準確度**
   - 依賴圖片品質
   - 手寫文字準確度較低
   - 複雜排版可能需要手動調整

2. **隱私和安全**
   - 確保圖片不包含敏感資訊
   - OCR 內容會被保存在 Markdown 中
   - 考慮資料安全需求

3. **檔案大小**
   - 添加 OCR 內容會增加 Markdown 檔案大小
   - 但遠小於 base64 inline 圖片
   - 仍然適合版本控制

## 🔧 技術細節

### Claude Code 工具使用

```python
# 步驟 1: 讀取圖片
Read(file_path="media/image.png")

# 步驟 2: Claude 視覺分析（自動）
# Claude 會自動識別這是圖片並分析內容

# 步驟 3: 編輯 Markdown
Edit(
    file_path="document.md",
    old_string="![title](media/image.png)",
    new_string="![title](media/image.png)\n\n<details>...[OCR content]...</details>"
)
```

### Pandoc 命令

```bash
# 基本轉換
pandoc input.docx -o output.md --extract-media=./media

# 進階選項
pandoc input.docx \
  -o output.md \
  --extract-media=./media \
  --wrap=none \
  --standalone
```
